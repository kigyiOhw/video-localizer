## What is Video-Localizer?

Video-Localizer is an AI-powered toolkit for managing video subtitles and audio tracks. Core workflow: **extract audio → ASR → generate subtitles**; **extract text → TTS → generate dubbing**; remux with FFmpeg.

> Subtitles and dubbing are generated by AI on the fly — no pre-existing files required.

## Features

- 🔍 **Stream Detection** — FFprobe-powered media inspection (video/audio/subtitle streams)
- 📤 **Stream Extraction** — Extract individual streams from any container
- 🎙️ **ASR** — Speech-to-text via faster-whisper (CUDA-accelerated, 5-tier hardware profiles)
- 🌐 **Translation** — LLM-powered subtitle translation (OpenAI-compatible API + Ollama local)
- 📝 **Subtitle Mux** — Add soft subtitles (SRT) to MKV/MP4 without re-encoding
- 🔗 **Pipeline** — End-to-end workflow: video → ASR → translate → mux (single page)
- ⚡ **SSE Streaming** — Real-time progress for ASR, translation, and pipeline operations
- 🖥️ **Web UI** — Jinja2 + HTMX server-rendered pages, zero frontend build step

## Architecture

```
Video File
  │
  ├─→ [Extract Audio] ──→ [ASR] ──→ SRT ──→ [Translate] ──→ Multi-lang SRT
  │
  ├─→ [Extract Text] ──→ [TTS] ──→ Audio ──→ [Align] ──→ New Audio Track
  │
  └─→ [FFmpeg Mux] ──→ Output Video (multi-sub + multi-audio tracks)
```

### Strategy Pattern

Each AI module follows a **strategy pattern** with swappable implementations:

```
engines/
├── asr/          # Speech recognition
│   ├── engine.py           # Abstract base (ASREngine)
│   └── whisper_local.py    # faster-whisper (CTranslate2, CUDA)
├── translate/    # Translation
│   ├── engine.py           # Abstract base (TranslateEngine)
│   └── llm.py              # LLM API + Ollama local
└── tts/          # Text-to-speech (planned)
    ├── engine.py
    └── edge_tts.py
```

## Quick Start

### Prerequisites

- **Python** ≥ 3.14
- **FFmpeg** ≥ 7.0 (with ffprobe)
- **CUDA-capable GPU** (optional — for GPU-accelerated ASR)

### Docker (Recommended)

```bash
# Clone
git clone https://github.com/kigyiOhw/video-localizer.git
cd video-localizer

# Configure
cp .env.example .env          # Edit media paths
# Optional: create config/settings.local.yaml for personal overrides

# Start
docker compose up -d
# → http://localhost:8000
```

### Direct Run

```bash
# Install dependencies
pip install -r requirements.txt

# Configure (required for translation)
# 默认 translate.engine 为 none，需手动启用：
#   cp config/settings.yaml config/settings.local.yaml
#   # 编辑 settings.local.yaml，设置 translate.engine 为 llm 或 llm_local
cp config/settings.yaml config/settings.local.yaml

# Start
python app.py
# → http://localhost:8000
```

> **注意**：翻译功能默认禁用。首次使用 `/translate` 或 `/pipeline` 前，
> 请在 `config/settings.local.yaml` 中将 `translate.engine` 设为 `llm`（OpenAI 兼容 API）
> 或 `llm_local`（本地 Ollama），并配置对应 API 密钥/地址。

For GPU-accelerated ASR, run the worker on the host machine:

```bash
python worker.py
# → http://localhost:9001 (CUDA inference endpoint)
```

## 项目简介

Video-Localizer 是一个 AI 驱动的视频字幕与音轨管理工具。核心工作流：**提取音频 → ASR 语音识别 → 生成字幕**；**提取文本 → TTS 语音合成 → 生成配音**；通过 FFmpeg 封装输出。

> 字幕和配音均由 AI 实时生成，不依赖现成文件。

## 功能特性

- 🔍 **流探测** — 基于 FFprobe 的媒体流检测（视频/音频/字幕流）
- 📤 **流提取** — 从任意容器中提取独立流
- 🎙️ **语音识别** — 基于 faster-whisper 的语音转文字（CUDA 加速，5 档硬件配置）
- 🌐 **翻译** — LLM 驱动的字幕翻译（支持 OpenAI 兼容 API + Ollama 本地运行）
- 📝 **字幕封装** — 添加软字幕（SRT）到 MKV/MP4，无需重新编码
- 🔗 **端到端流水线** — 视频 → ASR → 翻译 → 封装，一站式完成
- ⚡ **SSE 流式推送** — ASR、翻译、流水线实时进度
- 🖥️ **Web 界面** — Jinja2 + HTMX 服务端渲染，零前端构建

## 系统架构

```
视频文件
  │
  ├─→ [提取音频] ──→ [语音识别] ──→ 原文SRT ──→ [翻译] ──→ 多语SRT
  │
  ├─→ [提取文本] ──→ [语音合成] ──→ 音频 ──→ [对齐] ──→ 新音轨
  │
  └─→ [FFmpeg 封装] ──→ 输出视频（多字幕轨 + 多音轨）
```

### 策略模式

每个 AI 模块采用**策略模式**，实现可替换：

```
engines/
├── asr/          # 语音识别
│   ├── engine.py           # 抽象基类 (ASREngine)
│   └── whisper_local.py    # faster-whisper 实现 (CTranslate2, CUDA)
├── translate/    # 翻译
│   ├── engine.py           # 抽象基类 (TranslateEngine)
│   └── llm.py              # LLM API + Ollama 本地实现
└── tts/          # 语音合成（计划中）
    ├── engine.py
    └── edge_tts.py
```

## 快速开始

- **Python** ≥ 3.14
- **FFmpeg** ≥ 7.0（含 ffprobe）
- **CUDA GPU**（可选 — GPU 加速 ASR 需要）

### Docker（推荐）

```bash

cd video-localizer

# 配置
cp .env.example .env              # 编辑媒体路径

docker compose up -d

```

### 直接运行

```bash

pip install -r requirements.txt

# 启动
python app.py
# → http://localhost:8000
```

