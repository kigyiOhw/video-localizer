# AI 语音合成（TTS）与配音方案

## 概述

TTS（Text-to-Speech）将字幕文本合成为语音，实现给视频添加新语言配音的功能。

## 按场景推荐

| 场景 | 推荐引擎 | 理由 |
|------|----------|------|
| 桌面应用/免费 | **Edge-TTS** 或 **Coqui XTTS-v2** | 前者免费高质量，后者可声音克隆 |
| 中文配音天花板 | **GPT-SoVITS** 或 **CosyVoice 2** | 少样本即可克隆声音 |
| 多语种配音 | **XTTS-v2** | 支持 17 种语言 + 声音克隆 |
| Web 应用/商用 | **Azure TTS** | 最全语言 + 最佳质量 |
| 快速原型 | **Edge-TTS** | 零配置，微软免费服务 |

## 配音合成流程

```
字幕文件 (SRT/ASS)
     │
     ├─→ 提取文本 + 时间戳
     │
     ├─→ （可选）翻译为目标语言
     │
     ├─→ 按句子分段
     │
     ├─→ TTS 逐句合成音频
     │
     ├─→ 时间轴对齐（rubberband 变速）
     │
     └─→ 合并音频 → 封装为视频新音轨
```

---

## 时间轴对齐 — 关键难点

TTS 生成的音频长度 ≠ 原始语音长度，这是配音最棘手的部分。

### 对齐策略

#### 方案 A：按句子分段 TTS + 变速对齐（推荐）

```
原文: "Hello world" (1.2s)
TTS:  "你好世界"   (0.9s)
→ 拉伸 TTS 音频到 1.2s (rubberband/sox)
```

**关键库**：
- `rubberband` — 业界标准变速不变调
- `pyrubberband` — Python 封装
- `pydub` — 简易音频处理
- `ffmpeg atempo` — 基础变速（会变调）

#### 方案 B：调整视频播放速度

不自然，不推荐。

#### 方案 C：不严格对齐

允许音画偏差，适合旁白类内容，不适合对口型。

## 边缘 TTS（Edge-TTS）

微软免费 TTS 服务，质量高，零配置：

```python
import edge_tts

communicate = edge_tts.Communicate(
    "你好世界",
    "zh-CN-XiaoxiaoNeural"  # 微软晓晓-中文女声
)
await communicate.save("output.mp3")
```

- 优点：免费、高质量、多语种、零配置
- 缺点：需要网络、不可声音克隆

## 多模型并行能力

不同配置档的资源分配由 `config/requirements.py` 自动管理。GPU 显存有限时建议串行执行 ASR → 翻译 → TTS，或翻译用 CPU。

高配档位（`gpu_ultra`，≥16GB VRAM + 大内存）可同时驻留 TTS (XTTS) + ASR (large-v3-turbo)。

---

## 配音质量关键因素

| 因素 | 影响 | 建议 |
|------|------|------|
| TTS 引擎选择 | 声音自然度 | 中文优先 CosyVoice 2 / GPT-SoVITS |
| 分段粒度 | 节奏感 | 按句子分段，不按词 |
| 音频对齐 | 口型同步 | pyrubberband 逐句调整 |
| 声音克隆 | 一致性 | XTTS-v2 或 GPT-SoVITS |
| 背景音处理 | 纯净度 | 先分离人声（UVR/demucs）再合成 |
